## Feature Module Convention

Each feature under `src/features/` follows this structure (add only what's needed):

| Folder        | Contents                                                |
| ------------- | ------------------------------------------------------- |
| `model/`      | TypeScript types and Zod v4 schemas                     |
| `routes/`     | Route-level page components (default exports)           |
| `context/`    | React context providers and hooks                       |
| `utils/`      | Pure utility functions                                  |
| `components/` | Feature-scoped UI components (not shared)               |
| `api/`        | API call functions (for when real backend is connected) |
| `index.ts`    | Barrel  the public API for this feature           |file 

**Import rule:** Cross-feature imports must go through the feature's `index.ts` barrel only. Deep internal imports (e.g., `features/events/routes/EventFormPage`) are prohibited.

## Routing

Routes are defined in `src/App.tsx`. The router uses `basename="/live-events-dashboard"` to match the GitHub Pages deploy path.

| Path                    | Component              | Auth Required | Description                       |
| ----------------------- | ---------------------- | ------------- | --------------------------------- |
| `/login`                | `LoginPage`            | No            | Auth form                         |
| `/t/:tenantSlug/*`      |              | Path-based tenant override        |TenantSlugWrapper      | 
| `/e/:eventSlug`         | `EventLandingPage`     | No            | Public event page                 |
| `/`                     | `DashboardPage`        | Yes           | Client dashboard home             |
| `/events`               | `EventListPage`        | Yes           | Event list                        |
| `/events/new`           | `EventFormPage`        | Yes           | Create event                      |
| `/events/:id/edit`      | `EventFormPage`        | Yes           | Edit event                        |
| `/admin`                | `AdminDashboardPage`   | Yes           | Admin panel                       |
| `/settings`             | `SettingsPage`         | Yes           | User preferences                  |

## Data Layer

The app uses a **mock API** (`src/lib/api/mock/`) backed by `localStorage`. This lets the UI work independently of a backend.

**Migration path:** Create `src/lib/api/real/` with identical function signatures. Swap the export in `src/lib/api/index.ts`. Feature code needs no changes.

## Key Patterns

- **Zod v4 schemas:** `import { z } from "zod/ `z.url()` instead of `z.string().url()`v4"` 
- **No `zodResolver`:** Use `safeParse()` + manual `setError()` (resolver is incompatible with Zod v4)
- **No path aliases:** `@/` is not  use relative imports onlyconfigured 
- **UUID generation:** `crypto.randomUUID()` for all client-side entity IDs
- **Async cleanup:** `useEffect` hooks use a `cancelled` flag to prevent state updates after unmount
- **Stream URL security:** Embed URLs validated against HTTPS + provider whitelist in `streamUrl.ts`
